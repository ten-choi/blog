#### 변수, 클래스, 파일명 등 을 선언할 때 주의하면 좋은 점

#### 

가독성  
변수명만으로 어떤 기능을 가졌는지 알면 좋다  
접두어를 유용하게 사용하면 좋다  
혼동여부  
스코프가 작아야 중복적인 변수선언을 방지할 수 있다(전역 많으면 안 좋다)  
하나의 변수에 값을 이것저것 바꾸면 안 좋다(헷갈린다)  
스타일  
케멀케이스, 파스칼케이스 이런 거에 쓰임새를 class, interface 어디에 적용할지 미리정하자

#### 접두어 유형

#### 

-   is + 단어 : ~ 인지 아닌 boolean 값을 리턴  
    e.g) isVaild : 이것이 유효한지?
-   can + 단어 : ~ 가 가능한지 아닌지 boolean 값을 리턴  
    e.g) canRead : 읽을 수 있는지?(권한)
-   has + 단어 : ~ 가지고 있는지 여부 boolean 값을 리턴  
    e.g) hasItem : 그 가이템을 가지고 있는지?
-   on + 단어 : ~  boolean 값을 리턴  
    e.g) onClick : 버튼을 눌렀는지?

get, put, create, req, num, count, min, max, input, output, temp, dir  
prev, curr, next, last, init, def
