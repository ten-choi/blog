  🔴 Level 1 — TOC 명시 주제인데 중요 케이스가 빠진 것들

  2.2 인덱스 기본 사용법 — "탈 수 없는 조건" 중 몇 가지 누락

  현재는 선두 컬럼 실종 / OR / IN / 컬럼 가공 / 형변환 만 다룸. 누락:
  - NULL 조건 (IS NULL, IS NOT NULL) — 오라클 B-Tree 는 NULL 저장 안 해서 인덱스 못 탐
  - LIKE 전방/후방 일치 — 'abc%' 는 타고, '%abc' 는 못 탐
  - 부정형 조건 (!=, NOT IN, NOT EXISTS) — 인덱스 활용 제한적
  - BETWEEN / 범위 조건의 후행 컬럼 영향

  2.3 인덱스 확장 기능 사용법 — 제목은 "확장 기능"인데 현재는 스캔 방식만 다룸

  누락된 "진짜 확장 기능":
  - 함수 기반 인덱스 (Function-Based Index)
  - Invisible Index (통계만 남기고 실사용 차단)
  - Descending Index (DESC 정렬 전용 인덱스)
  - MySQL 의 Generated Column Index / Partial Index

  3.2 부분범위 처리 활용 — 개념만 3줄, 실전 예시 없음

  누락:
  - ROWNUM / LIMIT 활용 예제
  - Top-N 쿼리 패턴 (ORDER BY ... LIMIT N)
  - Pagination 튜닝 (OFFSET 안티패턴, 커서 기반 페이징)

  5.1 / 5.2 / 5.3 — 소트 전반이 깊이가 얕음

  - 5.1 소트 수행 원리: 정렬 알고리즘 언급 없음, Sort Area 관련 파라미터 (SORT_AREA_SIZE, PGA_AGGREGATE_TARGET, MySQL 의
   sort_buffer_size) 없음
  - 5.2 소트 안 발생 SQL: 현재 Union / Exists 두 케이스만. GROUP BY 의 해시 그룹핑 대체, Window 함수 정렬 생략 조건 등
  없음
  - 5.3 인덱스로 정렬 생략: 개념만. ORDER BY ... LIMIT N + 인덱스 조합, DESC 정렬 + Range Scan Descending, MIN/MAX
  최적화 예시 없음

  7.1 통계정보와 비용 계산 — 원리는 있는데 "어떻게 관리하느냐"가 빠짐

  - 통계 수집 방법 (오라클: DBMS_STATS.GATHER_TABLE_STATS, MySQL: ANALYZE TABLE)
  - 통계 자동 수집 스케줄 (오라클의 autotask)
  - Stale Statistics 문제와 재수집 기준
  - Dynamic Sampling — 통계 없는 테이블에 옵티마이저가 쓰는 대안

  🟡 Level 2 — 얕게 다뤄진 것들 (심화 보충 가치 있음)

  1.2 SQL 공유 및 재사용 — 바인드 변수 기본만 있음

  누락:
  - Bind Peeking — 첫 실행의 바인드 값을 참고해 계획 생성
  - Adaptive Cursor Sharing — 분포가 심한 경우 계획 여러 벌 유지
  - Cursor Sharing 파라미터 (CURSOR_SHARING=FORCE)

  4.1 NL 조인 — "튜닝 포인트"가 추상적

  누락:
  - Driving Table 선택 기준 (작은 쪽? 필터 강한 쪽?)
  - Batched NL Join (MySQL 8.0 BNL / 오라클의 NL JOIN BATCHING)
  - Inner 테이블 인덱스 요건

  4.3 해시 조인 — Build/Probe 만 있음

  누락:
  - Grace Hash Join vs In-Memory Hash Join 차이
  - Hash Area 부족 시 디스크 스왑 동작 원리 (파티션 단위 처리)

  4.4 서브쿼리 조인 — Unnesting 뒤에 어떻게 풀리는지가 모호

  누락:
  - Semi-Join / Anti-Join 명시적 개념 (EXISTS → Semi-Join, NOT EXISTS → Anti-Join)
  - Pushing Predicates

  6.2 Direct Path I/O — APPEND + NOLOGGING 위주

  누락:
  - Parallel DML 활성화 방법 (ALTER SESSION ENABLE PARALLEL DML)
  - SQL*Loader Direct 모드 옵션 예시

  6.3 파티션 활용 — Range / Hash / List 3종만

  누락:
  - Composite Partitioning (Range-Hash, Range-List 같은 조합)
  - Interval Partitioning (오라클 11g+, 자동 파티션 생성)
  - 파티션 Pruning 확인 방법 (실행 계획의 PARTITION RANGE SINGLE, KEY, ITERATOR 값 읽는 법)

  7.2 옵티마이저 힌트 — 힌트 사용법 중심

  누락:
  - SQL Plan Management (SPM) — 힌트 없이 플랜을 고정하는 방법
  - SQL Plan Baseline 개념
  - 힌트가 조용히 무시되는 구체 케이스 (alias 불일치 등)
